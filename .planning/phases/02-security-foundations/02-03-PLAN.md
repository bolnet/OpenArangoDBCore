LDAP authentication modes (from research):
- "simple": Construct DN = prefix + username + suffix, bind directly
- "search": Admin bind + ldap_search_ext_s to find user DN, then rebind as user

Role mapping methods:
- "roles attribute": Read roles from user's LDAP attribute (--ldap.roles-attribute-name)
- "roles search": Separate LDAP search for group membership (--ldap.roles-search)

CRITICAL: libldap LDAP* handles are NOT thread-safe. Per-request handles required.

    3. **Rewrite Enterprise/Auth/LDAPHandler.h**: Replace the empty stub. Follow Pattern 3 from research exactly:
       - `class LDAPHandler` with `explicit LDAPHandler(LDAPConfig config)`
       - `bool authenticate(std::string const& username, std::string const& password, std::vector<std::string>& outRoles)` -- main entry point
       - Private: `LDAP* createHandle()` -- creates fresh LDAP* per request via `ldap_initialize()`
       - Private: `void destroyHandle(LDAP* handle)` -- calls `ldap_unbind_ext()`
       - Private: `bool bindUser(LDAP* handle, std::string const& dn, std::string const& password)` -- `ldap_simple_bind_s()`
       - Private: `std::vector<std::string> fetchRoles(LDAP* handle, std::string const& userDn)` -- via attribute or search
       - CRITICAL: No shared LDAP* member. Each authenticate() call creates and destroys its own handle (Pitfall 3).

    4. **Implement Enterprise/Auth/LDAPHandler.cpp**:
       - `createHandle()`: Build URI string (`ldap://server:port` or `ldaps://server:port`). Call `ldap_initialize(&handle, uri.c_str())`. If `_config.useTLS`, call `ldap_start_tls_s(handle, nullptr, nullptr)` for StartTLS on port 389. If `_config.tlsCACertFile` is set, call `ldap_set_option(handle, LDAP_OPT_X_TLS_CACERTFILE, _config.tlsCACertFile.c_str())`.
       - `authenticate()` simple mode: `dn = prefix + username + suffix`, bindUser, fetchRoles, destroyHandle
       - `authenticate()` search mode: createHandle, admin bind (binddn/bindpasswd), `ldap_search_ext_s()` to find user DN, destroyHandle. Then createHandle again, bind as user with found DN, fetchRoles, destroyHandle.
       - `fetchRoles()` attribute mode: Read `_config.rolesAttribute` from user's LDAP entry via `ldap_search_ext_s()` + `ldap_get_values_len()`
       - `fetchRoles()` search mode: Search for groups where `member=userDn` using `_config.rolesSearch` filter
       - All errors: log via ArangoDB Logger, return false/empty

    5. **Make LDAP conditional in CMake**: Add `find_package(LDAP)` or use `pkg_check_modules(LDAP ldap)`. If not found, define `ARANGODB_HAVE_LDAP=0` and compile LDAPHandler.cpp as a stub that always returns false. This addresses research Open Question 3.
