<objective>
Implement RocksDB Encryption at Rest with AES-256-CTR

Purpose: Enable transparent encryption of all RocksDB SST and WAL files so that data on disk is never stored in plaintext. This is the most complex Phase 2 subsystem -- it implements the full `rocksdb::EncryptionProvider` abstract interface, key management (keyfile loading + rotation), and the `EncryptionFeature` ApplicationFeature lifecycle that hooks into RocksDB engine startup.

Output: Working EncryptionProvider that encrypts/decrypts via AES-256-CTR with per-file IVs, key management utilities, EncryptionFeature with option parsing and RocksDB integration, and comprehensive unit tests.
</objective>
