```cpp
#pragma once

#include <cstdint>
#include <string>

#include "Enterprise/RClone/CloudProvider.h"

namespace arangodb {

/// Immutable configuration for a cloud backup upload.
/// Constructed once from CLI options, never mutated.
struct RCloneConfig {
  CloudProvider provider;
  std::string endpoint;          // e.g. "s3.amazonaws.com" or custom
  std::string bucket;            // bucket or container name
  std::string pathPrefix;        // sub-path within bucket
  std::string region;            // provider region (S3/GCS)
  std::string rcloneBinaryPath;  // path to rclone executable
  uint64_t timeoutSeconds;       // per-transfer IO timeout (default 300)
  uint32_t transfers;            // parallel transfer count (default 4)
  uint32_t retries;              // retry count on failure (default 3)

  // Credential fields — stored in memory, injected as env vars, never serialized.
  std::string accessKeyId;       // S3
  std::string secretAccessKey;   // S3
  std::string azureAccount;      // Azure
  std::string azureKey;          // Azure
  std::string gcsServiceAccountFile; // GCS

  /// Validate that required fields are populated for the chosen provider.
  /// Returns empty string on success, error description on failure.
  [[nodiscard]] std::string validate() const;
};

}  // namespace arangodb
```

`validate()` checks:
- `rcloneBinaryPath` is non-empty
- `bucket` is non-empty
- For `kS3`: `accessKeyId` and `secretAccessKey` non-empty
- For `kAzureBlob`: `azureAccount` and `azureKey` non-empty
- For `kGCS`: `gcsServiceAccountFile` non-empty

1. **RCloneProcess.h** (new file):

```cpp
#pragma once

#include <cstdint>
#include <functional>
#include <optional>
#include <string>
#include <unordered_map>
#include <vector>

#include "Enterprise/RClone/RCloneConfig.h"

namespace arangodb {

/// Result of an rclone subprocess execution.
struct RCloneResult {
  int exitCode;
  std::string stdoutOutput;
  std::string stderrOutput;
  bool timedOut;
};

/// Progress callback: receives percentage (0-100).
using RCloneProgressCallback = std::function<void(uint32_t percent)>;

class RCloneProcess {
 public:
  /// Build the rclone command-line arguments for a copy operation.
  /// Does NOT include credentials — those go in the environment.
  static std::vector<std::string> buildCommand(
      RCloneConfig const& config,
      std::string const& localPath);

  /// Build the environment variable map for credential injection.
  /// All credentials are injected here, never as CLI args.
  static std::unordered_map<std::string, std::string> buildEnvironment(
      RCloneConfig const& config);

  /// Parse rclone progress output line and extract percentage.
  static std::optional<uint32_t> parseProgressLine(
      std::string_view line);

  /// Execute rclone subprocess synchronously.
  /// Calls progressCallback on each progress line if provided.
  /// Returns result with exit code, captured output, and timeout flag.
  static RCloneResult execute(
      RCloneConfig const& config,
      std::string const& localPath,
      RCloneProgressCallback progressCallback = nullptr);

 private:
  /// Fork and exec rclone with the given args and environment.
  /// Monitors stdout for progress lines.
  /// Enforces config.timeoutSeconds as IO idle timeout.
  static RCloneResult forkAndExec(
      std::vector<std::string> const& args,
      std::unordered_map<std::string, std::string> const& env,
      uint64_t timeoutSeconds,
      RCloneProgressCallback const& progressCallback);
};

}  // namespace arangodb
```

2. **RCloneProcess.cpp** (new file, ~200 lines):

Key implementation details:
- `buildCommand()`: Returns `[rcloneBinaryPath, "copy", localPath, "remote:" + bucket + "/" + pathPrefix, "--progress", "--stats", "1s", "--transfers", std::to_string(transfers), "--retries", std::to_string(retries), "--timeout", std::to_string(timeoutSeconds) + "s", "--log-level", "INFO"]`
- `buildEnvironment()`: Sets `RCLONE_CONFIG_REMOTE_TYPE`, then provider-specific vars. Uses `RCLONE_CONFIG_REMOTE_` prefix for all. Never includes credentials in the return value of `buildCommand()`.
- `parseProgressLine()`: Regex match on `Transferred:\s+[\d.]+ \w+ / [\d.]+ \w+, (\d+)%` to extract percentage integer.
- `forkAndExec()`:
  - Uses `posix_spawn()` or `fork()/execve()` with pipe for stdout capture
  - Inherits current env, overlays credential env vars
  - Reads stdout line by line, calls `parseProgressLine()` on each
  - Uses `select()`/`poll()` with timeout for IO idle detection
  - On timeout: sends SIGTERM, waits 5s, then SIGKILL
  - Returns `RCloneResult` with captured output

### Pitfall Guard:
- NEVER pass credentials as command-line arguments (visible in /proc/cmdline)
- Always check `waitpid()` return for EINTR
- Pipe fds must be closed in parent after dup2 in child
- Use `RCLONE_CONFIG_REMOTE_` prefix (not `RCLONE_S3_` or similar)

```cpp
#pragma once

#include <memory>
#include <string>
#include <string_view>

#include "ApplicationFeatures/ApplicationFeature.h"
#include "Enterprise/RClone/RCloneConfig.h"
#include "Enterprise/RClone/RCloneProcess.h"

namespace arangodb {

namespace options {
class ProgramOptions;
}

class RCloneFeature final
    : public application_features::ApplicationFeature {
 public:
  static constexpr std::string_view name() noexcept { return "RClone"; }

  explicit RCloneFeature(application_features::ApplicationServer& server);

  void collectOptions(std::shared_ptr<options::ProgramOptions>) override;
  void validateOptions(std::shared_ptr<options::ProgramOptions>) override;
  void prepare() override;
  void start() override;
  void beginShutdown() override;
  void stop() override;
  void unprepare() override;

  /// Upload a local backup directory to the configured cloud destination.
  /// Returns RCloneResult with exit code and captured output.
  /// progressCallback is invoked with percentage (0-100) during upload.
  RCloneResult uploadBackup(
      std::string const& localBackupPath,
      RCloneProgressCallback progressCallback = nullptr) const;

  /// Returns true if cloud backup is configured and ready.
  bool isConfigured() const noexcept;

  /// Access the current configuration (immutable after validateOptions).
  RCloneConfig const& config() const noexcept;

 private:
  // CLI option targets (populated by collectOptions/validateOptions)
  std::string _rcloneExecutable = "rclone";
  std::string _providerString;
  std::string _endpoint;
  std::string _bucket;
  std::string _pathPrefix;
  std::string _region;
  uint64_t _timeoutSeconds = 300;
  uint64_t _transfers = 4;
  uint64_t _retries = 3;

  // Built during validateOptions, immutable thereafter
  std::optional<RCloneConfig> _config;
  bool _configured = false;
  std::string _validationError;
};

}  // namespace arangodb
```

2. **RCloneFeature.cpp** (rewrite existing, ~180 lines):

- `collectOptions()`: Register all `--rclone-*` options binding to member variables:
  - `--rclone-executable` (string, default "rclone")
  - `--rclone-provider` (string, required: "s3", "azure", "gcs")
  - `--rclone-endpoint` (string)
  - `--rclone-bucket` (string)
  - `--rclone-path-prefix` (string, default "")
  - `--rclone-region` (string, default "")
  - `--rclone-timeout` (uint64_t, default 300)
  - `--rclone-transfers` (uint64_t, default 4)
  - `--rclone-retries` (uint64_t, default 3)
- `validateOptions()`:
  - Parse `_providerString` via `parseCloudProvider()`. If nullopt, set `_validationError`.
  - Build `RCloneConfig` from member vars. Credentials read from environment:
    - S3: `getenv("ARANGO_RCLONE_S3_ACCESS_KEY_ID")`, `getenv("ARANGO_RCLONE_S3_SECRET_ACCESS_KEY")`
    - Azure: `getenv("ARANGO_RCLONE_AZURE_ACCOUNT")`, `getenv("ARANGO_RCLONE_AZURE_KEY")`
    - GCS: `getenv("ARANGO_RCLONE_GCS_SERVICE_ACCOUNT_FILE")`
  - Call `_config->validate()`. If non-empty error, set `_validationError`.
  - Set `_configured = true` on success.
- `prepare()`: Verify rclone binary exists at `_rcloneExecutable` path (stat check).
- `start()`: Log readiness message with provider and bucket.
- `beginShutdown()` / `stop()` / `unprepare()`: No-op (uploads are synchronous per-call).
- `uploadBackup()`: Delegates to `RCloneProcess::execute(_config.value(), localBackupPath, progressCallback)`.
- `isConfigured()`: Returns `_configured`.
- `config()`: Returns `_config.value()`.

Credential flow:
1. User sets `ARANGO_RCLONE_S3_ACCESS_KEY_ID` etc. in shell environment
2. `validateOptions()` reads them into `RCloneConfig` struct (in-memory only)
3. `RCloneProcess::buildEnvironment()` maps them to `RCLONE_CONFIG_REMOTE_*` vars
4. `forkAndExec()` passes env vars to subprocess
5. Credentials never appear in CLI args, logs, or config files

### Pitfalls to Avoid

1. **Credentials in CLI args**: rclone supports `--s3-access-key-id` flags, but these are visible in `/proc/<pid>/cmdline`. Always use `RCLONE_CONFIG_REMOTE_*` environment variables instead.
2. **Credential logging**: Never log RCloneConfig fields that contain secrets. Implement a `toString()` that redacts credential fields.
3. **Pipe fd leaks**: When using `fork()`/`pipe()`, close the write end in the parent and the read end in the child. Failure to close causes the parent's `read()` to block indefinitely.
4. **EINTR on waitpid**: `waitpid()` can be interrupted by signals. Always retry on EINTR.
5. **Zombie processes**: If the parent fails to `waitpid()` after timeout-killing the child, the child becomes a zombie. Always reap.
6. **rclone not found**: `prepare()` must verify the binary exists. Provide a clear error message pointing to `--rclone-executable` option.
7. **Immutability violation**: `RCloneConfig` must not be modified after `validateOptions()` builds it. Use `std::optional<RCloneConfig>` and construct once.
8. **Environment variable namespace collision**: Use `ARANGO_RCLONE_*` prefix for user-facing env vars (input to ArangoDB), and `RCLONE_CONFIG_REMOTE_*` prefix for subprocess env vars (output to rclone). Do not confuse the two namespaces.
9. **Large backup directories**: rclone handles directory recursion natively. Do not manually enumerate files — pass the backup directory path directly to `rclone copy`.
10. **Timeout semantics**: `--timeout` in rclone is an IO idle timeout (not total transfer time). For total upload timeout, use the `select()`/`poll()` timeout in `forkAndExec()` separately.
