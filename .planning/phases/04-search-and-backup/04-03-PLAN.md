### 2.1 Class Declaration

```cpp
namespace arangodb {

class RocksDBHotBackup {
 public:
  struct CreateResult {
    BackupManifest manifest;
    double durationSeconds;
  };

  explicit RocksDBHotBackup(std::string backupBasePath,
                            rocksdb::DB* db = nullptr);

  // Create a new hot backup via RocksDB checkpoint
  Result create(std::string const& label, CreateResult& out);

  // List all available backups
  Result list(std::vector<BackupManifest>& out) const;

  // Delete a specific backup by ID
  Result deleteBackup(std::string const& backupId);

  // Restore a backup (validates manifest, copies checkpoint back)
  Result restore(std::string const& backupId);

  // Generate a unique backup ID based on current timestamp
  static std::string generateBackupId();

  // Get the backup directory path for a given ID
  std::string backupPath(std::string const& backupId) const;

 private:
  std::string _backupBasePath;   // e.g., "<dataDir>/backups"
  rocksdb::DB* _db;              // RocksDB instance (nullable for testing)
  mutable std::mutex _mutex;     // Protects concurrent backup operations

  // Internal: flush WAL, create checkpoint, write manifest
  Result createCheckpoint(std::string const& backupDir,
                          uint64_t& sequenceNumber);

  // Internal: collect collection names from the database
  Result collectCollectionNames(std::vector<std::string>& names) const;

  // Internal: recursively remove a directory
  static Result removeDirectory(std::string const& path);

  // Internal: validate backup directory integrity
  static Result validateBackupDirectory(std::string const& backupDir);
};

}  // namespace arangodb
```

### 2.3 Implementation (GREEN)

**RocksDBHotBackup.cpp** — implement the following sequence for `create()`:

```
1. generateBackupId() → backupId
2. Construct backupDir = _backupBasePath / backupId
3. Create backupDir on filesystem
4. GlobalWriteLock lock(_mutex)
5. _db->FlushWAL(true)                     // sync WAL to disk
6. Checkpoint::Create(_db, &checkpoint)     // get checkpoint handle
7. checkpoint->CreateCheckpoint(backupDir)  // hard-link SST files
8. lock released (RAII)
9. collectCollectionNames() → names
10. Build BackupManifest { backupId, timestamp, version, sequenceNumber, names, backupDir }
11. BackupManifest::saveToFile(backupDir + "/MANIFEST.json", manifest)
12. Return CreateResult { manifest, elapsed }
```

For `list()`:
```
1. Iterate subdirectories of _backupBasePath
2. For each subdir, attempt BackupManifest::loadFromFile()
3. Skip dirs with missing/corrupt manifests
4. Sort by timestamp ascending
5. Return vector of manifests
```

For `deleteBackup()`:
```
1. Construct backupDir = _backupBasePath / backupId
2. Verify directory exists
3. removeDirectory(backupDir) recursively
4. Return OK or error
```

For `restore()`:
```
1. Construct backupDir = _backupBasePath / backupId
2. validateBackupDirectory(backupDir) — check MANIFEST.json + SST files exist
3. Load manifest, verify version compatibility
4. Return OK (actual DB restart with checkpoint is orchestrated by caller)
```

### 2.4 Pitfall Guard
- Pitfall: Lock held too long — release BEFORE writing manifest (manifest write is not critical section)
- Pitfall: Partial checkpoint on crash — use two-phase: create in temp dir, rename atomically
- Pitfall: Hard-link across filesystems fails — backup dir MUST be on same filesystem as data dir

## Pitfalls to Avoid

| # | Pitfall | Impact | Mitigation |
|---|---------|--------|-----------|
