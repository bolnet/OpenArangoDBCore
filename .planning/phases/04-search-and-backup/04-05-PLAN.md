Community RocksDBIndex::fillIndex() interface:
  virtual Result fillIndex(transaction::Methods& trx,
                           std::shared_ptr<std::function<void()>> setRecoveryCallback);

```cpp
#pragma once
#ifndef ARANGODB_KEY_SPACE_PARTITIONER_H
#define ARANGODB_KEY_SPACE_PARTITIONER_H

#include <cstdint>
#include <string>
#include <vector>

namespace arangodb {

struct KeyRange {
  std::string lowerBound;  // inclusive
  std::string upperBound;  // exclusive (empty = unbounded)
};

class KeySpacePartitioner {
 public:
  /// Split the key space [lowerBound, upperBound) into numPartitions
  /// non-overlapping ranges. If upperBound is empty, it means unbounded.
  static std::vector<KeyRange> partition(
      std::string const& lowerBound,
      std::string const& upperBound,
      uint32_t numPartitions);

  /// Compute partition boundaries by sampling keys from an iterator.
  /// Returns numPartitions-1 split points.
  static std::vector<std::string> computeSplitPoints(
      std::vector<std::string> const& sampledKeys,
      uint32_t numPartitions);
};

}  // namespace arangodb
#endif
```

2. **KeySpacePartitioner.cpp** (new file):
   - `partition()`: If numPartitions == 1, return single range. Otherwise compute
     split points by dividing the key space evenly (byte-level interpolation) or
     by using sampled keys for data-aware partitioning.
   - `computeSplitPoints()`: Sort sampled keys, pick evenly spaced split points.

### Pitfall Guard:
- Key space is byte-ordered (RocksDB comparator), not lexicographic-aware for Unicode
- Empty upper bound means scan to end of column family

#ifndef ARANGODB_CHANGELOG_BUFFER_H
#define ARANGODB_CHANGELOG_BUFFER_H

#include <cstdint>
#include <functional>
#include <mutex>
#include <string>
#include <vector>

namespace arangodb {

enum class ChangelogOpType : uint8_t {
  kInsert = 0,
  kUpdate = 1,
  kDelete = 2
};

struct ChangelogEntry {
  ChangelogOpType opType;
  std::string documentKey;
  std::string documentRev;
  std::string documentData;  // empty for deletes
};

class ChangelogBuffer {
 public:
  explicit ChangelogBuffer(uint64_t memoryBudgetBytes);

  /// Append an operation. Returns false if memory budget exceeded.
  bool append(ChangelogEntry entry);

  /// Iterate all entries in insertion order.
  void forEach(std::function<void(ChangelogEntry const&)> const& cb) const;

  /// Number of entries buffered.
  uint64_t size() const;

  /// Approximate memory usage in bytes.
  uint64_t memoryUsage() const;

  /// Remove all entries.
  void clear();

 private:
  mutable std::mutex _mutex;
  std::vector<ChangelogEntry> _entries;
  uint64_t _memoryBudgetBytes;
  uint64_t _currentMemoryUsage;
};

}  // namespace arangodb
#endif
```

#pragma once
#ifndef ARANGODB_INDEX_BUILDER_THREAD_POOL_H
#define ARANGODB_INDEX_BUILDER_THREAD_POOL_H

#include <atomic>
#include <condition_variable>
#include <cstdint>
#include <functional>
#include <future>
#include <mutex>
#include <queue>
#include <thread>
#include <vector>

namespace arangodb {

class IndexBuilderThreadPool {
 public:
  explicit IndexBuilderThreadPool(uint32_t numThreads);
  ~IndexBuilderThreadPool();

  /// Submit a task. Returns a future for the Result.
  /// Returns std::nullopt if pool is shut down.
  std::optional<std::future<bool>> submit(std::function<bool()> task);

  /// Gracefully shut down: finish running tasks, reject new ones.
  void shutdown();

  /// Number of worker threads.
  uint32_t numThreads() const;

  /// Whether the pool is accepting new tasks.
  bool isRunning() const;

 private:
  void workerLoop();

  std::vector<std::thread> _workers;
  std::queue<std::packaged_task<bool()>> _taskQueue;
  std::mutex _mutex;
  std::condition_variable _cv;
  std::atomic<bool> _shutdown;
};

}  // namespace arangodb
#endif
```

2. **IndexBuilderThreadPool.cpp** (new file):
   - Constructor: Spawn numThreads workers, each running `workerLoop()`.
   - `workerLoop()`: Wait on condition variable, pop task from queue, execute.
   - `submit()`: If shutdown, return nullopt. Otherwise enqueue task, notify one worker.
   - `shutdown()`: Set flag, notify all, join all threads.
   - Destructor calls `shutdown()` if not already called.

### Pitfall Guard:

1. **RocksDBBuilderIndexEE.h** (modify existing):
```cpp
#pragma once
#ifndef ARANGODB_ROCKSDB_BUILDER_INDEX_EE_H
#define ARANGODB_ROCKSDB_BUILDER_INDEX_EE_H

#include "Enterprise/RocksDBEngine/ChangelogBuffer.h"
#include "Enterprise/RocksDBEngine/IndexBuilderThreadPool.h"
#include "Enterprise/RocksDBEngine/KeySpacePartitioner.h"

#include <atomic>
#include <cstdint>
#include <functional>
#include <memory>
#include <string>
#include <vector>

namespace arangodb {

/// Configuration for parallel index building.
struct IndexBuilderConfig {
  uint32_t numThreads = 4;
  uint64_t memoryBudgetMB = 256;
  bool backgroundMode = false;
};

/// Enterprise parallel index builder.
/// Wraps/extends Community RocksDBIndex::fillIndex() with:
///   - Key space partitioning across configurable thread pool
///   - Per-thread RocksDB iterators at same MVCC snapshot
///   - Background mode with shared lock and changelog buffer
///   - Atomic index swap on completion
class RocksDBBuilderIndexEE {
 public:
  explicit RocksDBBuilderIndexEE(IndexBuilderConfig config);
  ~RocksDBBuilderIndexEE();

  /// Fill the index in parallel. This is the enterprise replacement for
  /// the community single-threaded RocksDBIndex::fillIndex().
  ///
  /// @param snapshotFactory  Creates a per-thread iterator bound to the
  ///                         shared MVCC snapshot.
  /// @param indexInserter    Callback to insert a key-value pair into the
  ///                         index (thread-safe or partitioned).
  /// @param keyRangeLower    Lower bound of the key space (inclusive).
  /// @param keyRangeUpper    Upper bound of the key space (exclusive, empty=unbounded).
  /// @return true on success, false on failure.
  bool fillIndexParallel(
      std::function<std::unique_ptr<class IndexIterator>(
          std::string const& lowerBound,
          std::string const& upperBound)> snapshotFactory,
      std::function<bool(std::string const& key,
                         std::string const& value)> indexInserter,
      std::string const& keyRangeLower,
      std::string const& keyRangeUpper);

  /// Register a concurrent write that occurred during background build.
  /// Only valid when backgroundMode is true.
  bool bufferWrite(ChangelogEntry entry);

  /// Apply buffered changelog to the index after parallel fill completes.
  bool applyChangelog(
      std::function<bool(ChangelogEntry const&)> const& applier);

  /// Perform atomic swap: publish the newly built index and discard the old.
  /// Returns false if the build did not complete successfully.
  bool atomicSwap(std::function<void()> swapCallback);

  /// Query the current build state.
  enum class BuildState : uint8_t {
    kIdle = 0,
    kBuilding = 1,
    kApplyingChangelog = 2,
    kSwapping = 3,
    kComplete = 4,
    kFailed = 5
  };
  BuildState state() const;

  /// Access the configuration.
  IndexBuilderConfig const& config() const;

 private:
  IndexBuilderConfig _config;
  std::unique_ptr<IndexBuilderThreadPool> _threadPool;
  std::unique_ptr<ChangelogBuffer> _changelog;
  std::atomic<BuildState> _state;
};

/// Abstract iterator interface for index building.
/// Each thread gets its own iterator bound to the shared snapshot.
class IndexIterator {
 public:
  virtual ~IndexIterator() = default;
  virtual void seek(std::string const& key) = 0;
  virtual bool valid() const = 0;
  virtual void next() = 0;
  virtual std::string key() const = 0;
  virtual std::string value() const = 0;
};

}  // namespace arangodb
#endif  // ARANGODB_ROCKSDB_BUILDER_INDEX_EE_H
```

2. **RocksDBBuilderIndexEE.cpp** (modify existing):
   - Constructor: Initialize thread pool with config.numThreads, changelog buffer
     with config.memoryBudgetMB * 1024 * 1024.
   - `fillIndexParallel()`:
     1. Set state to kBuilding
     2. Call `KeySpacePartitioner::partition(lower, upper, numThreads)` to get ranges
     3. For each partition, submit task to thread pool:
        - Create iterator via snapshotFactory(range.lower, range.upper)
        - Seek to range.lower
        - Iterate until range.upper (or invalid), calling indexInserter for each entry
     4. Wait for all futures to complete
     5. If any task failed, set state to kFailed, return false
     6. Return true (state remains kBuilding until changelog applied)
   - `bufferWrite()`: Delegate to _changelog->append(entry). Return false if budget exceeded.
   - `applyChangelog()`:
     1. Set state to kApplyingChangelog
     2. Call _changelog->forEach() with the applier callback
     3. Clear changelog
   - `atomicSwap()`:
     1. Set state to kSwapping
     2. Call swapCallback
     3. Set state to kComplete

### Pitfall Guard:
- All threads MUST read at same MVCC snapshot version to avoid skew
- Final changelog apply is single-threaded (merge bottleneck for very large collections)
- Do not call CreateColumnFamily() without flush coordination (compaction interference)
- Changelog must track deletes via _rev field to handle delete-then-reinsert correctly

## Pitfalls to Avoid

| # | Pitfall | Mitigation |
|---|---------|-----------|
| 1 | Threads reading at different MVCC snapshots causing data skew | Snapshot is acquired once and shared; each thread gets an iterator bound to the same snapshot via snapshotFactory |
| 2 | CreateColumnFamily() + concurrent builds without flush coordination causes compaction interference | Do not create column families during build; coordinate with max_background_jobs |
| 3 | Changelog must track deletes via _rev field | ChangelogEntry includes opType and documentRev; apply phase handles delete-then-reinsert sequences correctly |
| 4 | Final merge/changelog apply is single-threaded (bottleneck) | Document this limitation; changelog should be small relative to full collection; future optimization can parallelize apply |
| 5 | Thread pool destructor must join threads or std::terminate is called | Destructor calls shutdown() which joins all threads |
| 6 | Mutex contention in ChangelogBuffer under high write load | Use per-shard or per-partition sub-buffers if contention is measured; initial implementation uses single mutex (sufficient for typical workloads) |
| 7 | Memory budget accounting is inaccurate | Track approximate usage conservatively: sizeof(ChangelogEntry) + key.capacity() + rev.capacity() + data.capacity() per entry |
| 8 | Key space partitioning produces uneven partitions (data skew) | Support sampling-based partitioning via computeSplitPoints() for data-aware distribution |
