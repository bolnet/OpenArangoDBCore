<objective>
Implement TopK/WAND optimizer rule for IResearch views

Purpose: Detect the EnumerateViewNode -> SortNode (BM25 desc) -> LimitNode pattern in AQL execution plans and enable WAND (Weak AND) early termination on the EnumerateViewNode. This allows IResearch to skip documents whose maximum possible BM25 score cannot compete with the current k-th best score, dramatically reducing the number of full score computations for top-k queries.

Output: AQL optimizer rule (optimize-top-k), WandIterator with early termination, ScoreThresholdManager with min-heap tracking, and 20 comprehensive tests.

```cpp
/// Tracks the top-k BM25 scores using a min-heap.
/// The threshold (minimum score in the heap) is the k-th best score seen.
/// Documents scoring below this threshold can be safely skipped.
class ScoreThresholdManager {
 public:
  explicit ScoreThresholdManager(size_t k);

  /// Submit a candidate score. Returns true if the score was competitive
  /// (inserted into the heap), false if it was below threshold.
  bool addScore(float score);

  /// Current threshold: 0.0 if fewer than k scores seen, otherwise the
  /// k-th best (minimum in the min-heap of top-k scores).
  float threshold() const;

  /// Number of scores currently tracked (max k).
  size_t size() const;

  /// Configured capacity.
  size_t capacity() const;

  /// Clear all tracked scores, reset threshold to 0.0.
  void reset();

 private:
  size_t _k;
  /// Min-heap: smallest element at top. When full, top = k-th best = threshold.
  std::vector<float> _heap;
};
```

2. **Enterprise/IResearch/IResearchOptimizeTopK.cpp** — implement:

```cpp
ScoreThresholdManager::ScoreThresholdManager(size_t k) : _k(k) {
  _heap.reserve(k);
}

bool ScoreThresholdManager::addScore(float score) {
  if (_heap.size() < _k) {
    _heap.push_back(score);
    std::push_heap(_heap.begin(), _heap.end(), std::greater<float>{});
    return true;
  }
  if (score > _heap.front()) {
    std::pop_heap(_heap.begin(), _heap.end(), std::greater<float>{});
    _heap.back() = score;
    std::push_heap(_heap.begin(), _heap.end(), std::greater<float>{});
    return true;
  }
  return false;
}

float ScoreThresholdManager::threshold() const {
  if (_heap.size() < _k) { return 0.0f; }
  return _heap.front();
}

size_t ScoreThresholdManager::size() const { return _heap.size(); }
size_t ScoreThresholdManager::capacity() const { return _k; }

void ScoreThresholdManager::reset() {
  _heap.clear();
}
```

```cpp
/// Abstract base for a scored document iterator (mirrors IResearch doc_iterator).
class ScoredDocIterator {
 public:
  virtual ~ScoredDocIterator() = default;
  virtual bool next() = 0;
  virtual uint64_t value() const = 0;
  virtual uint64_t seek(uint64_t target) = 0;
  virtual float score() = 0;
  virtual float maxScore() = 0;  // upper bound on remaining documents
};

/// WAND iterator that wraps a ScoredDocIterator and skips documents whose
/// max possible score is below the current competitive threshold.
/// This is the core early-termination mechanism for top-k BM25 queries.
class WandIterator : public ScoredDocIterator {
 public:
  /// @param inner  The underlying scored document iterator to wrap.
  /// @param manager  Shared threshold manager (updated externally as scores arrive).
  WandIterator(std::unique_ptr<ScoredDocIterator> inner,
               std::shared_ptr<ScoreThresholdManager> manager);

  bool next() override;
  uint64_t value() const override;
  uint64_t seek(uint64_t target) override;
  float score() override;
  float maxScore() override;

  /// Number of documents skipped due to threshold pruning.
  size_t skippedCount() const;

 private:
  std::unique_ptr<ScoredDocIterator> _inner;
  std::shared_ptr<ScoreThresholdManager> _manager;
  size_t _skipped = 0;

  /// Advance inner iterator past non-competitive documents.
  bool advancePastNonCompetitive();
};
```

2. **Enterprise/IResearch/IResearchOptimizeTopK.cpp** — implement:

```cpp
WandIterator::WandIterator(std::unique_ptr<ScoredDocIterator> inner,
                           std::shared_ptr<ScoreThresholdManager> manager)
    : _inner(std::move(inner)), _manager(std::move(manager)) {}

bool WandIterator::next() {
  while (_inner->next()) {
    if (_inner->maxScore() >= _manager->threshold()) {
      return true;  // potentially competitive, caller will score()
    }
    ++_skipped;
  }
  return false;
}

uint64_t WandIterator::value() const { return _inner->value(); }

uint64_t WandIterator::seek(uint64_t target) {
  auto docId = _inner->seek(target);
  // After seek, check competitiveness; if not, advance
  if (docId != 0 && _inner->maxScore() < _manager->threshold()) {
    if (!next()) { return 0; }
    return _inner->value();
  }
  return docId;
}

float WandIterator::score() {
  float s = _inner->score();
  _manager->addScore(s);  // update threshold with actual score
  return s;
}

float WandIterator::maxScore() { return _inner->maxScore(); }

size_t WandIterator::skippedCount() const { return _skipped; }
```

```cpp
bool isBM25SortElement(aql::SortElement const& element) {
  // BM25 scorer detected by attribute path containing "BM25" (case-insensitive)
  auto const& path = element.attributePath;
  std::string lower = path;
  std::transform(lower.begin(), lower.end(), lower.begin(), ::tolower);
  return lower.find("bm25") != std::string::npos;
}

size_t optimizeTopKPatterns(
    std::vector<aql::MockExecutionNode*> const& nodes) {
  size_t optimized = 0;

  for (auto* node : nodes) {
    // Look for LimitNode as the starting point (bottom-up traversal)
    if (node->getType() != aql::ExecutionNodeType::LIMIT) continue;

    auto* limitNode = static_cast<aql::MockLimitNode*>(node);
    if (limitNode->offset() != 0) continue;  // WAND requires offset=0
    if (limitNode->limit() == 0) continue;

    // Check parent is SortNode
    auto* parent = node->getFirstDep();
    if (!parent || parent->getType() != aql::ExecutionNodeType::SORT) continue;

    auto* sortNode = static_cast<aql::MockSortNode*>(parent);

    // Verify sort is BM25 descending
    auto const& elements = sortNode->elements();
    if (elements.empty()) continue;
    if (!isBM25SortElement(elements[0])) continue;
    if (elements[0].ascending) continue;  // must be DESC

    // Check sort's parent is EnumerateViewNode
    auto* grandparent = parent->getFirstDep();
    if (!grandparent ||
        grandparent->getType() != aql::ExecutionNodeType::ENUMERATE_IRESEARCH_VIEW) continue;

    auto* viewNode = static_cast<aql::MockEnumerateViewNode*>(grandparent);

    // Annotate the view node: enable WAND with heap size = limit
    viewNode->setWandEnabled(true);
    viewNode->setWandHeapSize(limitNode->limit());
    ++optimized;
  }

  return optimized;
}
```

<pitfalls>
## Pitfalls to Avoid

1. **Approximate results**: WAND must return exact top-k. The threshold is a lower bound — documents with maxScore() >= threshold MUST be fully scored. Only skip when maxScore() is strictly below threshold.

2. **Threshold equality edge case**: When a document's maxScore() equals the threshold exactly, it must NOT be skipped. Use strict less-than (`<`) for the skip condition, never less-than-or-equal.

3. **Empty heap threshold**: Before k documents have been scored, the threshold must be 0.0 (accept all documents). Returning a nonzero threshold early would incorrectly skip valid results.

4. **LimitNode with offset**: WAND only works for pure top-k (offset=0). A query like `LIMIT 5, 10` requires scoring the top 15 and then skipping 5 — the optimizer rule must reject this pattern (or handle it by setting heapSize = offset + limit, which adds complexity; avoid for v1).

5. **SortNode with multiple elements**: If the SortNode sorts by `[BM25 DESC, name ASC]`, the secondary sort does not affect WAND applicability — but only if the primary sort is BM25. Check `elements()[0]` specifically, not all elements.

6. **Mutating the execution plan**: The optimizer rule must only set `_enableWand` and `_wandHeapSize` on existing EnumerateViewNode. Creating new node types or removing existing nodes would break the plan tree invariants.

7. **Thread safety of ScoreThresholdManager**: In single-query execution, the manager is accessed from one thread. Do NOT add unnecessary mutex locks — they would degrade performance. Document the single-threaded assumption.

8. **maxScore() accuracy**: The correctness of WAND depends entirely on maxScore() being a true upper bound. If the underlying IResearch iterator returns an underestimate, results will be wrong. The WandIterator must trust but document this contract.

9. **Score type precision**: Use `float` (not `double`) to match IResearch's internal score representation. Mixing float/double in comparisons causes subtle threshold mismatches.

10. **Registration timing**: The optimizer rule must be registered during feature prepare() phase, before any query planning occurs. Late registration means queries bypass the optimization silently.
</pitfalls>
