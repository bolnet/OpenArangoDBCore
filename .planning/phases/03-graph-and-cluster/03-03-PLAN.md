### Tests (RED):
- LocalTraversalNodeTest.cpp:
  - `LocalTraversalNode_IsLocalGraphNode_ReturnsTrue`: isLocalGraphNode() returns true
  - `LocalTraversalNode_GetType_ReturnsTraversal`: NodeType is TRAVERSAL (same as standard)
  - `LocalTraversalNode_ToVelocyPack_WritesLocalFlag`: Serialized VelocyPack contains "isLocalGraphNode": true
  - `LocalTraversalNode_FromVelocyPack_RoundTrip`: Deserialize via createLocalGraphNode factory, verify isLocalGraphNode
  - `CreateLocalGraphNode_TraversalType_ReturnsLocalTraversal`: Factory creates LocalTraversalNode for TRAVERSAL type
  - `CreateLocalGraphNode_ShortestPathType_ReturnsLocalShortestPath`: Factory handles SHORTEST_PATH
  - `CreateLocalGraphNode_EnumeratePathsType_ReturnsLocalEnumeratePaths`: Factory handles ENUMERATE_PATHS
  - `LocalTraversalNode_Clone_PreservesLocalFlag`: Cloned node is still local
  - `IsUsedAsSatellite_SatelliteGraph_ReturnsTrue`: Enterprise override returns true for satellite graphs

### Implementation (GREEN):
1. **LocalGraphNode.h**: Shared mixin/base:
   - Stores `_isLocal = true`, `_isUsedAsSatellite`, `_clusterOneShardRuleEnabled`
   - Provides enterprise overrides for `isLocalGraphNode()`, `isUsedAsSatellite()`, `enableClusterOneShardRule()`

2. **LocalTraversalNode.h/.cpp**: Extends TraversalNode:
   - `isLocalGraphNode()` → true
   - `doToVelocyPack()` → writes `"isLocalGraphNode": true` in addition to base serialization
   - `createBlock()` → creates execution block for local traversal
   - `clone()` → returns new LocalTraversalNode preserving local flag
   - `estimateCost()` → lower cost than standard TraversalNode (no network hops)

3. **LocalEnumeratePathsNode.h**: Same pattern for ENUMERATE_PATHS
4. **LocalShortestPathNode.h**: Same pattern for SHORTEST_PATH

5. **createLocalGraphNode() factory** (in LocalTraversalNode.cpp or separate file):
   - Read NodeType from VelocyPack
   - Switch on type → create LocalTraversalNode, LocalShortestPathNode, or LocalEnumeratePathsNode
   - This replaces the community stub that throws TRI_ERROR_NOT_IMPLEMENTED

## Task 2: Implement SmartGraphProvider and SmartGraphStep

TDD: Write tests first for shard routing.

### Tests (RED):
- SmartGraphProviderTest.cpp:
  - `SmartGraphProvider_IsResponsible_LocalShard_ReturnsTrue`: Provider recognizes local shards
  - `SmartGraphProvider_IsResponsible_RemoteShard_ReturnsFalse`: Provider rejects remote shards
  - `SmartGraphProvider_StartVertex_RoutesToCorrectShard`: Given start vertex "eu:123", routes to shard holding "eu" prefix
  - `SmartGraphStep_IsLocal_LocalVertex_ReturnsTrue`: Step for local vertex is marked local
  - `SmartGraphStep_IsLocal_RemoteVertex_ReturnsFalse`: Step for remote vertex is not local
  - `PathValidatorEE_DisjointPath_SamePartition_Valid`: Path within same smartGraphAttribute partition passes
  - `PathValidatorEE_DisjointPath_CrossPartition_Invalid`: Path crossing partitions fails validation

### Implementation (GREEN):
1. **SmartGraphProvider.h/.cpp**: Template-compatible provider:
   - `isResponsible(VertexRef)` → check if vertex's smart prefix maps to local shard
   - `startVertex(VertexRef)` → resolve to local or remote, create initial step
   - `expand(Step)` → enumerate edges from local shard, create child steps
   - Uses shard map (collectionToShard) from GraphNode for routing decisions

2. **SmartGraphStep.h/.cpp**: Enterprise step type:
   - Stores: `_vertexId`, `_shardId`, `_isLocal` flag, `_smartPrefix`
   - `isLocal()` → true if step's shard is on this DB-Server
   - `getSmartPrefix()` → extract prefix for routing

3. **SmartGraphRPCCommunicator.h**: RPC stub for remote shard communication (declare interface, minimal impl)

4. **PathValidatorEE.cpp**: Enterprise path validation:
   - `checkValidDisjointPath(Step)` → verify step's vertex has same smart prefix as traversal start
   - Integrates with PathValidatorOptions `_isDisjoint` flag

5. **OneSidedEnumeratorEE.tpp**: Enterprise extension:
   - `smartSearch()` → dispatch to SmartGraphProvider for smart-aware neighbor enumeration
   - Conditional compilation: use SmartGraphStep when `is_smart` template parameter is true

### Pitfall Guard:
- LocalGraphNode must NOT introduce new NodeType values — reuse existing TRAVERSAL/SHORTEST_PATH/ENUMERATE_PATHS
- Template-based providers: SmartGraphProvider must match duck-typed interface exactly
- PathValidatorEE disjoint check: compare smart prefixes, not full vertex IDs
