SatelliteCollection configuration:
  replicationFactor: "satellite" → internally stored as 0
  numberOfShards: 1 (always)
  writeConcern: 0 (auto = all DB-Servers)

CollectionAccess satellite tracking (AQL layer):
  bool isUsedAsSatellite() const noexcept;
  void useAsSatelliteOf(ExecutionNodeId prototypeAccessId);
  std::optional<ExecutionNodeId> _isSatelliteOf;

Graph class satellite support:
  virtual bool isSatellite() const;
  uint64_t replicationFactor() const;
  std::unordered_set<std::string> const& satelliteCollections() const;

ShardLocking satellite logic:
  Satellite shards locked on ALL servers involved in the query.
  Satellite eligibility: col->isSatellite() && (pushToSingleServer || graphNode->isSmart())

SmartToSat edge validation:
  Edge shard key derived from SmartCollection vertex's smartGraphAttribute (not satellite side)
  _from/_to referencing SatelliteCollection permitted without matching smartGraphAttribute

## Task 1: Implement SatelliteDistribution

TDD: Write tests first for satellite shard assignment.

### Tests (RED):
- SatelliteDistributionTest.cpp:
  - `IsSatellite_ReplicationFactorZero_ReturnsTrue`: replicationFactor=0 detected as satellite
  - `IsSatellite_ReplicationFactorNonZero_ReturnsFalse`: replicationFactor=3 is not satellite
  - `AssignShards_SatelliteCollection_AllServers`: Given 5 DB-Servers, satellite shard is assigned to all 5
  - `AssignShards_SatelliteCollection_SingleShard`: Satellite always has exactly 1 shard
  - `WriteConcern_Satellite_EqualsServerCount`: Write concern for satellite = number of DB-Servers
  - `SatelliteCollectionSet_TracksRegistered`: Can register and query satellite collection names
  - `SmartToSat_EdgeValidation_SatelliteSideSkipsSmartCheck`: Edge with _to in satellite collection doesn't require smartGraphAttribute match on satellite side
  - `SmartToSat_EdgeValidation_SmartSideStillValidated`: Edge with _from in smart collection still requires correct _key prefix

### Implementation (GREEN):
1. **SatelliteDistribution.h**: Declare:
   - `class SatelliteDistribution` with:
     - `static bool isSatellite(uint64_t replicationFactor)` — returns true if replicationFactor == 0
     - `static uint64_t satelliteReplicationFactor()` — returns 0
     - `static uint32_t satelliteNumberOfShards()` — returns 1
     - `struct SatelliteShardAssignment`:
       - `ShardID shardId`
       - `std::vector<ServerID> servers` (all DB-Servers)
     - `static SatelliteShardAssignment assignShards(std::vector<ServerID> const& dbServers)`
     - `static uint64_t satelliteWriteConcern(size_t numServers)` — returns numServers
   - Satellite collection registry:
     - `class SatelliteCollectionRegistry`:
       - `void registerSatellite(std::string const& collectionName)`
       - `bool isSatelliteCollection(std::string const& name) const`
       - `std::unordered_set<std::string> const& all() const`

2. **SatelliteDistribution.cpp**: Implement all methods.
   - `assignShards()`: Create single shard, assign to all provided servers
   - First server in list becomes leader; rest are followers
   - Integration with SmartToSat: provide helper `isSmartToSatEdge()` that checks if either endpoint references a registered satellite collection

### Pitfall Guard:
- Satellite replicationFactor is 0, NOT the string "satellite" — that's the API representation only
- numberOfShards must ALWAYS be 1 for satellite collections
- Write concern = 0 in storage means "all servers" — do not confuse with "no write concern"
