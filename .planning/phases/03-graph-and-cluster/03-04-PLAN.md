## Task 1: Implement ReadFromFollower

TDD: Write tests first for replica selection.

### Tests (RED):
- ReadFromFollowerTest.cpp:
  - `GetServers_DirtyReadEnabled_IncludesFollowers`: With dirty-read, returns leader + followers
  - `GetServers_DirtyReadDisabled_OnlyLeader`: Without dirty-read, returns only leader
  - `RoundRobin_DistributesAcrossReplicas`: Over N calls, all replicas are selected approximately equally
  - `RoundRobin_SingleReplica_AlwaysReturnsLeader`: With replicationFactor=1, always returns leader
  - `WriteOperation_NeverRoutedToFollower`: Even with dirty-read header, write ops go to leader
  - `MultiShard_EachShardRoutedIndependently`: Different shards can be routed to different followers
  - `ResponseHeader_DirtyRead_SetsPotentialDirtyRead`: Response gets X-Arango-Potential-Dirty-Read when follower used
  - `ResponseHeader_LeaderRead_NoPotentialDirtyRead`: Response does NOT get dirty-read header when leader served the read
  - `StalenessConfig_BoundedStaleness_RejectsLaggingFollower`: Follower exceeding staleness bound is excluded
  - `StalenessConfig_EventualConsistency_AcceptsAnyFollower`: With eventual consistency, any follower is eligible
  - `StreamTransaction_HeaderOnCreate_AppliesToAllOps`: Dirty-read on txn creation applies to subsequent ops

### Implementation (GREEN):
1. **ReadFromFollower.h**: Declare:
   - `class ReadFromFollower`:
     - `struct ReplicaSelection`:
       - `ServerID serverId`
       - `bool isFollower` (true if not the leader)
     - `static void getResponsibleServersReadFromFollower(`
       `  containers::FlatHashSet<ShardID> const& shards,`
       `  containers::FlatHashMap<ShardID, ServerID>& result,`
       `  ShardToServerMap const& shardLeaderMap,`
       `  ShardToFollowersMap const& shardFollowersMap)`
     - `static ReplicaSelection chooseReplica(`
       `  ShardID const& shard,`
       `  ServerID const& leader,`
       `  std::vector<ServerID> const& followers)`
     - `static bool isDirtyReadAllowed(RequestHeaders const& headers)` — checks X-Arango-Allow-Dirty-Read
     - `static void setDirtyReadResponseHeader(ResponseHeaders& headers, bool followerUsed)` — sets X-Arango-Potential-Dirty-Read
   - Staleness configuration:
     - `struct StalenessConfig`:
       - `enum class Mode { Eventual, Bounded }`
       - `Mode mode = Mode::Eventual`
       - `std::chrono::milliseconds maxStaleness{0}` — 0 = eventual, >0 = bounded
     - `static bool isFollowerEligible(ServerID const& follower, StalenessConfig const& config)`

2. **ReadFromFollower.cpp**: Implement:
   - `getResponsibleServersReadFromFollower()`:
     - For each shard, collect leader + followers
     - Call `chooseReplica()` for each shard
     - Store selected server in result map
   - `chooseReplica()`:
     - Round-robin selection using thread-safe atomic counter
     - `static std::atomic<uint64_t> _roundRobinCounter{0}`
     - `index = _roundRobinCounter.fetch_add(1) % (1 + followers.size())`
     - index 0 = leader, index 1..N = followers[index-1]
   - `isDirtyReadAllowed()`:
     - Check for "X-Arango-Allow-Dirty-Read" header with value "true"
   - `setDirtyReadResponseHeader()`:
     - If follower was used, set "X-Arango-Potential-Dirty-Read: true"
   - `isFollowerEligible()`:
     - Eventual mode: always eligible
     - Bounded mode: check follower's reported lag against maxStaleness
     - If lag information unavailable, default to eligible (graceful degradation)

### Pitfall Guard:
- Round-robin counter must be atomic (concurrent coordinator threads)
- Never route writes to followers — check operation type before routing
- Stream Transactions: dirty-read flag is set once at creation, not per-operation
- When no followers are available (replicationFactor=1), always return leader without error
