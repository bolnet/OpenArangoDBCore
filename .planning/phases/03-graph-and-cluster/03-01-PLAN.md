## Task 1: Implement ShardingStrategyEE

TDD: Write tests first for shard routing correctness.

### Tests (RED):
- ShardingStrategyEETest.cpp:
  - `EnterpriseHashSmartEdge_RoutesByKeyPrefix`: Given edge _key "eu:us:12345", routes to shard based on "eu" prefix hash
  - `EnterpriseHashSmartEdge_SamePrefix_SameShard`: Edges with same _from prefix route to same shard
  - `EnterpriseHexSmartVertex_RoutesByPrefix`: Given vertex _key "eu:abc", routes by "eu" hash
  - `EnterpriseHexSmartVertex_NoColon_RoutesFullKey`: Key without colon uses full key as routing value
  - `AllStrategies_Name_ReturnsCorrectName`: Each strategy returns its registered name
  - `AllStrategies_UsesDefaultShardKeys_ReturnsFalse`: Enterprise strategies don't use default shard keys

### Implementation (GREEN):
1. **ShardingStrategyEE.h**: Declare three classes:
   - `EnterpriseHashSmartEdgeShardingStrategy : public ShardingStrategy`
   - `EnterpriseHexSmartVertexShardingStrategy : public ShardingStrategy`  
   - `EnterpriseSmartEdgeCompatShardingStrategy : public ShardingStrategy`
   - Helper: `extractSmartPrefix(std::string_view key) -> std::string_view`
   - Helper: `computeShardIndex(std::string_view value, uint32_t numberOfShards) -> uint32_t`

2. **ShardingStrategyEE.cpp**: Implement:
   - `extractSmartPrefix()`: Extract substring before first `:` from _key
   - `computeShardIndex()`: Consistent hash (CRC32 or FNV-1a, matching ArangoDB's hash) mod numberOfShards
   - `EnterpriseHashSmartEdgeShardingStrategy::getResponsibleShard()`:
     - Parse _key to extract first prefix (fromSmartValue)
     - Hash prefix → shard index
     - Return ShardID from shard list
   - `EnterpriseHexSmartVertexShardingStrategy::getResponsibleShard()`:
     - If _key contains `:`, extract prefix; else use full key
     - Hash → shard index → ShardID
   - Registration function: `registerEnterpriseShardingStrategies(ShardingFeature&)`

3. **Tests/mocks**: `ShardingMocks.h` with mock ShardingInfo providing shard lists.

### Pitfall Guard:
- Pitfall 6 (Immutable smartGraphAttribute): Enforce in schema, not strategy
- Hash function MUST match ArangoDB's internal hash for shard routing compatibility

## Task 2: Implement SmartGraphSchema

TDD: Write tests first for key format validation.

### Tests (RED):
- SmartGraphSchemaTest.cpp:
  - `ValidateKey_CorrectFormat_Accepts`: _key "eu:12345" with smartGraphAttribute "region" = "eu" passes
  - `ValidateKey_MissingColon_Rejects`: _key "12345" without colon is rejected
  - `ValidateKey_PrefixMismatch_Rejects`: _key "us:12345" but region = "eu" is rejected
  - `ValidateKey_MissingAttribute_Rejects`: Document without smartGraphAttribute is rejected
  - `ValidateEdge_DisjointConstraint_SamePartition_Accepts`: Edge from "eu:a" to "eu:b" in disjoint graph passes
  - `ValidateEdge_DisjointConstraint_CrossPartition_Rejects`: Edge from "eu:a" to "us:b" in disjoint graph is rejected
  - `ValidateEdge_NonDisjoint_CrossPartition_Accepts`: Cross-partition edge allowed in non-disjoint graph
  - `Immutability_CannotChangeSmartAttribute`: Attempting to modify smartGraphAttribute on update is rejected
  - `SmartToSat_EdgeWithSatelliteVertex_Accepts`: Edge referencing satellite collection vertex is valid

### Implementation (GREEN):
1. **SmartGraphSchema.h**: Declare:
   - `class SmartGraphSchema` with:
     - `static Result validateDocument(VPackSlice doc, std::string const& smartAttribute, bool isDisjoint)`
     - `static Result validateEdge(VPackSlice edge, std::string const& smartAttribute, bool isDisjoint, std::unordered_set<std::string> const& satelliteCollections)`
     - `static std::string_view extractSmartValue(std::string_view key)`
     - `static std::string buildSmartKey(std::string_view smartValue, std::string_view localKey)`
     - `static std::string buildSmartEdgeKey(std::string_view fromSmart, std::string_view toSmart, std::string_view edgeKey)`

2. **SmartGraphSchema.cpp**: Implement:
   - `validateDocument()`: Check _key contains `:`, prefix matches smartGraphAttribute value
   - `validateEdge()`: Check disjoint constraint (if isDisjoint, _from and _to must have same prefix)
   - `extractSmartValue()`: Return substring before first `:`
   - Smart-to-Satellite edge validation: if _to or _from references a satellite collection, skip smartGraphAttribute check for that side

3. **VirtualClusterSmartEdgeCollection.h**: Declare class that wraps:
   - `_local_E`: edges where both vertices are on this shard
   - `_from_E`: edges where _from vertex is on this shard
   - `_to_E`: edges where _to vertex is on this shard
   - Methods: `getLocalEdges()`, `getFromEdges()`, `getToEdges()`

### Pitfall Guard:
- Pitfall 6 check: `canModifyShardingAttribute()` returns false
- _key format MUST use `:` separator (not `/` or other)
- Empty smartGraphAttribute value is invalid
